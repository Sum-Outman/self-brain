# Self Brain AGI 中文使用指南

## 📖 项目简介

Self Brain 是一个革命性的自主人工智能系统，通过自学习、自优化和跨领域协作能力实现真正的通用人工智能（AGI）。系统集成了11个专业子模型（A-K），涵盖管理、语言、视觉、音频、推理等多个维度，共同在统一架构下协同工作。

---

## 🚀 快速开始

### 系统要求
- Python 3.8 或更高版本
- Windows/Linux/macOS 操作系统
- 至少 8GB 内存
- 至少 10GB 可用磁盘空间
- 推荐使用 Chrome、Firefox 或 Edge 浏览器
- 可选：外部摄像头、传感器和设备

### 安装步骤

1. **克隆仓库**
```bash
git clone https://github.com/Sum-Outman/self-brain.git
cd self-brain
```

2. **创建虚拟环境**
```bash
# Windows
python -m venv myenv
myenv\Scripts\activate

# Linux/Mac
source myenv/bin/activate
```

3. **安装依赖**
```bash
pip install -r requirements.txt
```

4. **启动系统**
```bash
# 方式1：启动完整系统（推荐）
python start_system.py

# 方式2：启动独立A管理模型
python a_manager_standalone.py

# 方式3：仅启动Web界面
python web_interface/app.py
```

5. **访问界面**
   启动后，在浏览器中访问：http://localhost:5000

## 🎯 核心功能

### 1. 聊天交互

与 A_management 主模型进行自然对话：
- 在主界面输入文本消息
- 支持多种语言交互
- 可接收文本、图像、音频等多种形式的响应
- 体验具有情感分析能力的智能对话

### 2. 知识库管理

创建和管理系统知识：
- **创建知识条目**：手动添加新的知识内容
- **导入知识**：批量导入外部知识源
- **备份知识**：定期备份知识库以防丢失
- **自主学习**：启动模型对知识库的自主学习（详见下文）
- **数据库维护**：清理和优化知识库数据库

### 3. 模型训练

训练和优化各个AI模型：
- **单独训练**：选择特定模型进行训练
- **联合训练**：同时训练多个相关模型
- **上传训练数据**：为模型提供外部训练素材
- **监控训练进度**：实时查看训练状态和效果
- **调整训练参数**：根据需要优化模型训练配置

### 4. 系统设置

配置系统和模型参数：
- **模型配置**：调整各模型的运行参数
- **API集成**：设置外部API连接（详见下文）
- **设备配置**：配置摄像头和传感器（详见下文）
- **系统偏好**：设置界面和运行偏好

### 5. 实时监控

跟踪系统和模型性能：
- **系统指标**：查看CPU、内存、网络等系统资源使用情况
- **模型状态**：监控各模型的健康状态和运行指标
- **日志查看**：访问系统和模型日志进行故障排查

## 🧠 知识库自主学习功能

Self Brain 的知识库自主学习功能允许各个模型针对知识库内容进行自主学习，提升其专业能力。

### 使用步骤

1. 导航到**知识库管理**页面
2. 在**自主学习控制**区域，从下拉菜单中选择：
   - **All Models**：让所有模型同时进行学习
   - 特定模型（A-K）：选择单个模型进行学习
3. 点击**Start Self-Learning**按钮开始学习
4. 观察实时进度条和状态信息
5. 需要时点击**Stop Self-Learning**按钮停止学习

### 学习原理

每个模型会根据自身特性和领域，从知识库中筛选相关内容进行学习：
- **A管理模型**：学习管理策略、系统协调和人机交互相关知识
- **B语言模型**：学习语言模式、语法规则、情感表达相关知识
- **C音频模型**：学习音频处理、语音识别、声音合成相关知识
- **D图像模型**：学习图像处理、图像识别、特征提取相关知识
- **E视频模型**：学习视频处理、动作识别、内容分析相关知识
- **F空间模型**：学习空间感知、3D建模、定位导航相关知识
- **G传感器模型**：学习传感器数据处理、信号分析相关知识
- **H计算机控制模型**：学习系统控制、自动化操作相关知识
- **I运动模型**：学习运动控制、执行器管理相关知识
- **J知识库模型**：学习各类学科知识、专业理论相关知识
- **K编程模型**：学习编程知识、代码优化、算法设计相关知识

## 🌐 外部API集成

Self Brain 支持每个模型独立接入外部API服务，以补充或替代本地模型功能。

### 配置步骤

1. 导航到**设置**页面
2. 找到**模型API配置**部分
3. 选择要配置的模型（A-K）
4. 填写以下信息：
   - **API URL**：外部模型服务的接口地址
   - **API Key**：访问外部服务的认证密钥
   - **模型名称**：在外部服务中使用的具体模型名称
5. 点击**测试连接**按钮验证配置是否正确
6. 保存配置以应用更改

### 支持的外部模型类型

系统设计兼容主流AI服务提供商：
- 大型语言模型（如OpenAI GPT系列、Anthropic Claude等）
- 图像生成和识别模型
- 音频处理和语音识别服务
- 视频分析服务
- 代码生成和分析工具

## 🔌 外部设备与传感器集成

Self Brain 提供了标准化接口，支持连接多种外部设备和传感器。

### 支持的设备类型

- **摄像头**：用于视觉感知（支持多摄像头以实现双目视觉）
- **温湿度传感器**：用于环境监测
- **运动传感器**：如加速度计、陀螺仪等
- **位置传感器**：如GPS模块、距离传感器
- **环境传感器**：如光照、气压、烟雾传感器
- **执行器**：如电机控制器、继电器模块

### 配置方法

1. 确保设备正确连接到计算机
2. 导航到**设置**页面的**设备与传感器**部分
3. 根据设备类型选择相应的配置选项
4. 填写必要的连接参数（如端口号、通信速率等）
5. 保存配置并重启相关服务

### 多摄像头配置

为支持双目视觉功能，系统允许配置多个摄像头：
1. 将多个摄像头连接到计算机
2. 在系统配置文件 `config/system_config.yaml` 中设置 `camera_sources` 数组
   ```yaml
   camera_sources: [0, 1]  # 使用第一个和第二个摄像头
   ```
3. 重启系统以应用配置

## 🛠️ 故障排除

### 常见问题与解决方案

1. **端口冲突**
   - 症状：启动时出现"Address already in use"错误
   - 解决：修改 `config/system_config.yaml` 中的端口配置，或关闭占用端口的程序

2. **内存不足**
   - 症状：系统运行缓慢或崩溃
   - 解决：关闭其他程序以释放内存，减少并发任务数量，避免同时训练多个大型模型

3. **API连接失败**
   - 症状：外部API调用失败
   - 解决：检查API URL、密钥是否正确，确认网络连接正常，验证API服务状态

4. **设备无法识别**
   - 症状：连接的设备或传感器不被系统识别
   - 解决：检查设备连接，更新驱动程序，确认端口配置正确

## 📚 模型详细介绍

Self Brain 包含11个专业子模型，每个模型负责特定领域的任务：

### A_management（管理模型）
中央协调器，具有情感分析能力，管理所有其他模型和人机交互。

### B_language（语言模型）
多语言交互模型，具备情感推理能力。

### C_audio（音频模型）
音频处理模型，能够进行语音识别、语调分析、语音合成、音乐识别和噪音处理。

### D_image（图像模型）
图像处理模型，用于内容识别、编辑、增强和语义图像生成。

### E_video（视频模型）
视频处理模型，用于内容识别、编辑、修改和语义视频生成。

### F_spatial（空间模型）
双目空间感知模型，用于3D空间识别、建模、定位、距离感知和运动预测。

### G_sensor（传感器模型）
传感器数据处理模型，支持温度、湿度、加速度、陀螺仪、压力、光等各种传感器。

### H_computer_control（计算机控制模型）
计算机系统控制模型，具有多系统兼容性，用于自动化操作。

### I_motion（运动模型）
运动和执行器控制模型，用于复杂控制任务，具有多端口输出和通信协议兼容性。

### J_knowledge（知识库模型）
核心知识专家模型，具备物理、数学、化学、医学、法律、历史、心理学、工程学等多领域的综合知识。

### K_programming（编程模型）
编程辅助模型，能够自主生成代码和进行系统自我改进。
